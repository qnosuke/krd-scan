# UI刷新（ライトテーマ）+ 推移グラフ・前回比表示 設計書

日付: 2026-07-15
状態: ユーザー承認済み

## 目的

読み取り機能が実用レベルに達したので、次の3点で使い勝手を上げる。

1. 実機（KRD-203-W: 白ボディ+淡いグレーの液晶）のイメージに近い、柔らかい配色への刷新
2. 体重など7項目の推移を見られるグラフ
3. 前回測定との差の表示（例: 前回より -0.5kg）

## ユーザー確認済みの決定

- 配色: **白ベース+液晶グリーン**（アイボリー暖色案・端末設定追従案は不採用）
- グラフ対象: **全7項目をチップで切替**（体重のみ案は不採用）
- 置き場所: **履歴タブを刷新**（サマリーカード → グラフ → 従来リストの縦積み。グラフ専用タブ新設・リストタップ詳細案は不採用）
- ダークモードは作らない（単一ライトテーマ）

## 制約（変更不可）

- 外部ライブラリ・外部送信・トラッキングを追加しない（情報タブのプライバシー約束を維持）
- IndexedDB名 'karadascan'、DBスキーマ、認識エンジン（sevenseg.js）、項目割当（session.js）には触らない

## 設計

### 1. 配色刷新（styles.css 全面改訂）

`:root` のCSS変数を差し替える。変数経由で全画面に効く構造は既存のまま。

- `--bg`: ごく薄い暖かみのあるグレー（実機の白ボディの印象）
- `--panel`: 白。柔らかい影を追加
- 数値表示部分（チップ・サマリーカード・確認画面の値）: 淡いグレーグリーンの「液晶風」背景 + 濃いグリーンの数字
- `--accent`: 落ち着いた青緑（テール系）。ボタン・アクティブタブに使用
- タブバー: ライト配色のすりガラス風（backdrop-filter維持）
- カメラ画面のガイド枠: 視認性優先で色を調整
- `index.html` の `theme-color` メタ、PWAマニフェストの `theme_color`/`background_color` も更新

### 2. 履歴タブの新構成（上から順）

1. **サマリーカード**: 最新測定の日時 + 全7項目の最新値と前回比をグリッド表示。体重は大きめ。
   - 前回比は「▲0.3 / ▼0.5 / ±0」のニュートラル表示。**色分けしない**
     （体重は減が良くても骨格筋率は増が良い、と項目で価値が逆のため）
   - 前回値がない項目（記録1件のみ・前回が手入力スキップ等）は前回比を出さない
2. **推移グラフ**: 自作インラインSVG折れ線
   - 7項目のチップで切替（既存チップUIの見た目を流用）
   - 期間切替: 「1ヶ月 / 3ヶ月 / 全部」の3ボタン（初期値は1ヶ月）
   - 最新点に値ラベル。タップ操作等のインタラクションは初版では作らない
   - Y軸はデータのmin/maxに余白を付けた自動スケール
   - データ点が多い場合は間引いて描画
   - 選択期間にデータが2点未満なら「データがたまるとグラフが表示されます」等の案内
3. **履歴リスト**: 現行のまま（配色のみ刷新）+ CSVエクスポートボタン

### 3. 確認画面（保存前）に前回値表示

各項目の入力行に「前回 63.2」を薄く表示する。前回比が見えるだけでなく、
前回と大きく違う値は誤読の可能性があるというチェックにも効く。

### 4. 実装構造

- 新規 `src/chart.js`: **純関数**。レコード配列 + 項目キー + 期間 → SVG描画用の座標データ
  （スケール計算・間引き・ラベル位置を含む）。DOMに触らない
- 新規 `src/trend.js`: 前回比計算の純関数（historyView と confirmView の両方から使う）
- `src/ui/historyView.js`: サマリーカード + グラフのDOM構築を追加
- `src/ui/confirmView.js`: 前回値の薄表示を追加
- `src/styles.css`: 全面改訂
- DB・CSV・認識エンジン・セッションは変更なし

### 5. テスト・検証

- vitest: chart.js のスケール計算・間引き・期間フィルタ、trend.js の前回比計算
- ローカルブラウザ（Playwright）で表示確認: 履歴0件 / 1件 / 複数件の各状態
- デプロイ後、iPhone実機でユーザー確認
