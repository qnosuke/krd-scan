# 目標設定・日次サマリー・手動入力 設計

2026-07-17 承認済み。KRD scan に以下の3機能をまとめて追加する。

1. 🎯 目標設定と達成表示 — 体重・体脂肪率に目標値を設定し、履歴画面に目標との差と達成バッジを表示
2. 📅 日次サマリービュー — 同日複数測定を日単位でグループ化し、代表値（その日の最新）で表示
3. 🧮 測定記録の手動入力 — カメラなしで全項目を手入力するフォーム（確認画面を流用、日時も指定可能）

## 確定した要件（ユーザー回答）

- 3機能まとめて1設計・1プラン
- 日次サマリーの代表値 = **その日の最新値**（平均ではない）
- 目標の達成判定 = **最新値 ≦ 目標値で達成**（減量前提。方向設定は作らない）
- 手動入力の導線 = **計測タブ**のボタン
- 目標の設定・変更 = **履歴タブの目標カード**内で完結
- 採用アプローチ = 案A「軽量拡張」（localStorage の目標 + 日次グループ化 + 確認画面流用。IndexedDB 無変更・新依存なし）

## 全体の制約

- IndexedDB は一切変更しない（DB_NAME='karadascan'、DB_VERSION=1、既存レコード完全互換）
- 新しい実行時依存を追加しない
- プライバシー約束（端末内のみ・外部送信なし・解析なし）を維持する。localStorage も端末内のみなので影響なし
- `src/sevenseg.js`・`src/session.js` には触らない

## 1. 目標値のデータと保存（新規 `src/goals.js`）

- 保存先: **localStorage**、キー `krdscan-goals`
- 形式: JSON オブジェクト `{ "weight": "63.0", "bodyFat": "20.0" }`
  - DB の測定値と同じく**文字列**で持つ。未設定の項目はキーなし
  - 対象は体重（weight）と体脂肪率（bodyFat）の2項目のみ。片方だけの設定も可
- 関数:
  - `loadGoals()` → goals オブジェクト。localStorage が空・壊れた JSON・不正値のときは `{}` を返す
  - `saveGoals(goals)` → localStorage へ書き込み。空オブジェクトならキーごと削除
  - `goalStatus(latestRaw, goalRaw)` → `{ diff, achieved }` の純関数
    - `achieved` = `Number(latestRaw) <= Number(goalRaw)`
    - `diff` = `Number(latestRaw) - Number(goalRaw)`（正なら未達で「あと diff」、負・0 なら達成）
    - どちらかが null / 数値化できないときは `null` を返す
- 記録が1件もない、または該当項目が未測定（null）の場合は差分を計算しない（目標値のみ表示）

## 2. 目標カード UI（履歴タブ）

- 位置: サマリーカード（最新の測定）の直下、推移グラフカードの上
- 表示（設定済みの項目ごとに1行）:
  - 未達成: `体重 目標 63.0kg ・ あと 1.2kg`
  - 達成: `体重 目標 63.0kg ・ 🎉 達成！（−0.4kg）`
  - 達成バッジは常時表示。一過性のポップアップやトーストは作らない
  - 最新値がない項目は目標値のみ表示（差分なし）
- 編集: カード右上の「編集」でカード内がその場のフォームに変わる（体重・体脂肪率の数値入力2つ + 保存/キャンセル）
  - 入力の検証は既存 `validateInput`（metrics.js）を流用。範囲外はアラートで弾く
  - 空にして保存 = その項目の目標を解除
- 目標が未設定（両方ともなし）のときは「目標を設定する」ボタンだけの小さなカードを表示

## 3. 日次サマリー（新規 `src/daily.js`）

- 純関数 `groupByDay(records)`:
  - 入力は `listMeasurements()` の戻り値（新しい順）
  - **ローカルタイムゾーン**の日付（YYYY-MM-DD）でグループ化
  - 戻り値: 新しい日順の配列 `[{ dateKey, records（新しい順）, latest（その日の最新レコード） }]`
- 純関数 `dailyLatest(records)`:
  - 1日1レコード（その日の最新）に間引いた配列を返す（グラフ用）
- UI への反映（`src/ui/historyView.js`）:
  - **履歴リスト**: フラット一覧 → 日付見出し付きグループ表示。見出しは `2026/07/17（木）・2件` 形式（曜日付き、件数は常に表示）。レコード行は現状のまま（時刻・値・削除ボタン）
  - **今日の測定状態**: サマリーカード上部に `✓ 今日は測定済み` / `今日はまだ測定していません` のチップを表示
  - **推移グラフ**: `buildChart` に渡す records を `dailyLatest(records)` に変更（1日1点）。`buildChart` 自体は無変更
- **前回比（computeDelta）は現状のまま**、直前レコードとの比較。同日2回目の測定では「前回＝同日の前の測定」との比較になるが、確認画面の「前回」表示と挙動が揃うため変更しない

## 4. 測定記録の手動入力

- 導線: **計測タブ**のボタン列（読み取り開始など）の下に控えめなリンク「カメラを使わず手入力で記録する」を追加（リンク自体は `index.html`、クリック時の画面遷移は `src/main.js` で配線。`captureView.js` は変更しない）
- タップで既存の**確認画面**（view-confirm）を空の結果で開く（手入力モード）
- 手入力モードのときだけ:
  - 見出し・説明文を手入力向けの文言に出し分け
  - 一覧の先頭に**日時入力欄**（`<input type="datetime-local">`、初期値 = 現在時刻）を表示
  - 保存時は入力された日時を `measuredAt` に使う（不正な日時はアラートで弾く）
- カメラ / 動画取り込み経由の既存フローは**従来どおり**（日時欄なし・`measuredAt` = 保存時刻）
- 値の検証（validateInput・範囲外アラート）、「前回 N」表示、全項目空のガード、保存処理（addMeasurement）はすべて既存のものをそのまま使う

## エラー処理

- localStorage 読み込み失敗・壊れた JSON → 目標未設定として扱う（例外を外に漏らさない）
- 目標入力の範囲外 → `validateInput` でアラート（既存の確認画面と同じ挙動）
- 手入力の日時が空・不正 → アラートで保存を止める
- 記録ゼロ件でも目標カードは表示可能（目標値のみ。差分なし）

## やらないこと（YAGNI）

- 目標の方向設定（以上で達成）・目標履歴・体重/体脂肪率以外の目標
- 日次代表値の「平均」オプション、全測定/日次のグラフ切替
- 達成時の一過性ポップアップ・通知
- IndexedDB スキーマ変更
